# 백엔드 서버 실행 가이드

GIS 지도에서 프로젝트 데이터를 불러오려면 Flask 백엔드 서버가 실행되어야 합니다.

## 문제 상황

현재 `http://127.0.0.1:5500`에서 접속하고 있는데, 이는 정적 파일 서버(Live Server 등)입니다.
Flask 백엔드 서버는 별도로 실행해야 합니다.

## 해결 방법

### 방법 1: Flask 서버 실행 (권장)

1. **터미널을 엽니다**

2. **백엔드 디렉토리로 이동:**
   ```bash
   cd backend
   ```

3. **Flask 서버 실행:**
   ```bash
   python app.py
   ```
   또는
   ```bash
   flask run
   ```

4. **서버가 실행되면:**
   - Flask 서버: `http://127.0.0.1:5000`
   - 정적 파일 서버: `http://127.0.0.1:5500` (현재 사용 중)

5. **API 요청이 자동으로 Flask 서버로 전달됩니다**

### 방법 2: Flask 서버에서 직접 프론트엔드 제공

1. **Flask 서버 실행:**
   ```bash
   cd backend
   python app.py
   ```

2. **브라우저에서 접속:**
   ```
   http://127.0.0.1:5000/gis.html
   ```

이 경우 정적 파일 서버는 필요 없습니다.

## 확인 방법

### Flask 서버가 정상 실행되었는지 확인:

브라우저에서 다음 URL을 열어보세요:
```
http://127.0.0.1:5000/api/health
```

**정상적인 응답:**
```json
{
  "status": "healthy",
  "service": "GBMS",
  "version": "1.0.0"
}
```

### GIS API 확인:

```
http://127.0.0.1:5000/api/gis/projects
```

**로그인이 필요한 경우:**
- 먼저 `http://127.0.0.1:5000`에서 로그인
- 또는 인증 토큰이 필요할 수 있습니다

## 문제 해결

### "Module not found" 오류

필요한 패키지를 설치하세요:
```bash
cd backend
pip install -r requirements.txt
```

### 포트가 이미 사용 중

다른 포트로 실행:
```bash
python app.py --port 5001
```

그리고 `frontend/js/api.js`에서 포트를 수정:
```javascript
BASE_URL: 'http://127.0.0.1:5001/api'
```

### CORS 오류

`backend/config.py`에서 CORS 설정을 확인하세요.

## 빠른 테스트

터미널에서 다음 명령어로 서버를 실행:

```bash
cd backend
python app.py
```

서버가 실행되면:
- 콘솔에 "Running on http://127.0.0.1:5000" 메시지가 표시됩니다
- 브라우저에서 `http://127.0.0.1:5000/api/health`를 열어 확인하세요
