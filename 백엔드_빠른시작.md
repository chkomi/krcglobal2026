# 백엔드 서버 빠른 시작 가이드

## 문제: API 404 오류

현재 `/api/gis/projects` 요청이 404를 반환하고 있습니다.
이는 Flask 백엔드 서버가 실행되지 않았기 때문입니다.

## 해결 방법

### 1단계: Flask 설치 (처음 한 번만)

```bash
cd backend
pip install -r requirements.txt
```

또는

```bash
pip install Flask Flask-SQLAlchemy Flask-CORS PyJWT Werkzeug python-dotenv
```

### 2단계: 백엔드 서버 실행

터미널에서:

```bash
cd backend
python app.py
```

또는

```bash
cd backend
python3 app.py
```

### 3단계: 서버 확인

서버가 정상 실행되면 다음과 같은 메시지가 표시됩니다:

```
 * Running on http://127.0.0.1:5000
 * Debug mode: on
```

### 4단계: 브라우저에서 확인

1. **Health Check:**
   ```
   http://127.0.0.1:5000/api/health
   ```
   응답: `{"status": "healthy", ...}`

2. **GIS API (인증 필요할 수 있음):**
   ```
   http://127.0.0.1:5000/api/gis/projects
   ```

## 현재 설정

- **정적 파일 서버**: `http://127.0.0.1:5500` (Live Server)
- **Flask 백엔드 서버**: `http://127.0.0.1:5000` (실행 필요)

API 요청은 자동으로 Flask 서버(5000)로 전달됩니다.

## 문제 해결

### "Module not found: flask"

```bash
pip install Flask
```

### "Address already in use"

다른 프로그램이 포트 5000을 사용 중입니다.
포트를 변경하거나 해당 프로그램을 종료하세요.

### "No such table: projects"

데이터베이스를 초기화하세요:

```bash
cd backend
python init_db.py
```

## 개발 모드

현재 GIS API는 인증을 임시로 비활성화했습니다 (개발용).
프로덕션 환경에서는 인증을 다시 활성화하세요.
